  <action>
Create backend/src/controllers/profileController.js with two exported async handler functions.

**getGithubProfile(req, res, next):**
1. Read username from req.params.username
2. Set headers: { Authorization: `token ${process.env.GITHUB_TOKEN}`, 'User-Agent': 'devtrackr' }
3. Fetch user: `https://api.github.com/users/${username}` — extract login, name, bio, avatar_url, public_repos, followers, html_url
4. Fetch repos: `https://api.github.com/users/${username}/repos?per_page=100` — sum up stargazers_count across all repos to get total_stars
5. If GitHub returns 404, respond with res.status(404).json({ error: 'GitHub user not found' })
6. If network error (catch on the fetch), respond with res.status(502).json({ error: 'Failed to reach GitHub API' })
7. On success, respond 200 with: { login, name, bio, avatar_url, public_repos, total_stars, followers, html_url }

**getLeetcodeProfile(req, res, next):**
1. Read username from req.params.username
2. POST to https://leetcode.com/graphql with headers: { 'Content-Type': 'application/json', 'Referer': 'https://leetcode.com' }
3. Body (JSON.stringify): { query: GRAPHQL_QUERY_STRING, variables: { username } }
4. GraphQL query string (define as a const at top of file):
```
query getUserProfile($username: String!) {
  matchedUser(username: $username) {
    username
    submitStats: submitStatsGlobal {
      acSubmissionNum {
        difficulty
        count
        submissions
      }
    }
    profile {
      ranking
      reputation
    }
  }
}
```
5. Parse response JSON. If matchedUser is null, return res.status(404).json({ error: 'LeetCode user not found' })
6. From acSubmissionNum array, find entries where difficulty is 'Easy', 'Medium', 'Hard', and 'All'. Use the 'All' entry for acceptance_rate = Math.round((All.count / All.submissions) * 100) — handle division by zero (submissions=0 → rate=0).
7. Return 200 with: { username, easy_solved, medium_solved, hard_solved, total_solved, acceptance_rate, ranking }
8. Network errors → res.status(502).json({ error: 'Failed to reach LeetCode API' })

Both functions use outer try/catch → next(err) for unexpected errors, matching the pattern in apiTesterController.js.

Do NOT use require('../db') — this controller makes no database queries.
