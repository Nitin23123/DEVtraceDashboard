Create frontend/src/api/profile.js following the exact pattern of dashboard.js:

```js
const API_URL = process.env.REACT_APP_API_URL || 'http://localhost:5000';

export const getGithubProfile = async (token, username) => {
  const res = await fetch(`${API_URL}/api/profile/github/${encodeURIComponent(username)}`, {
    headers: { 'Authorization': `Bearer ${token}` }
  });
  return res.json();
};

export const getLeetcodeProfile = async (token, username) => {
  const res = await fetch(`${API_URL}/api/profile/leetcode/${encodeURIComponent(username)}`, {
    headers: { 'Authorization': `Bearer ${token}` }
  });
  return res.json();
};
```

Use encodeURIComponent on the username to handle edge cases.

**frontend/src/pages/ProfilePage.js:**

Create the page component. Structure:
- State: githubUsername (string), leetcodeUsername (string), githubData (object|null), leetcodeData (object|null), githubLoading (bool), leetcodeLoading (bool), githubError (string), leetcodeError (string)
- On mount: read localStorage.getItem('github_username') and localStorage.getItem('leetcode_username') and pre-fill the state variables
- loadGithub handler:
  1. setGithubLoading(true), setGithubError('')
  2. call getGithubProfile(token, githubUsername)
  3. if result.error → setGithubError(result.error)
  4. else → setGithubData(result), localStorage.setItem('github_username', githubUsername)
  5. setGithubLoading(false)
- loadLeetcode handler: same pattern with getLeetcodeProfile and 'leetcode_username' key

Layout — two widgets side by side using flexbox:
```jsx
<div style={{ display: 'flex', gap: '24px', flexWrap: 'wrap', alignItems: 'flex-start' }}>
  {/* GitHub widget — left */}
  {/* LeetCode widget — right */}
</div>
```

Each widget is a card div (white bg, border, border-radius 12px, padding 24px, minWidth 280px, flex: 1).

**GitHub widget contents (when githubData is loaded):**
- Avatar: <img src={githubData.avatar_url} alt="avatar" style={{ width: 64, height: 64, borderRadius: '50%' }} />
- Name: h3 (githubData.name or githubData.login)
- Bio: p style={{ color: '#64748b' }} — only if bio is non-null
- Stats grid (use a simple definition list or divs):
  - Public Repos: {githubData.public_repos}
  - Total Stars: {githubData.total_stars}
  - Followers: {githubData.followers}
- Link: <a href={githubData.html_url} target="_blank" rel="noreferrer" style={{ color: '#3b82f6' }}>View on GitHub</a>

**LeetCode widget contents (when leetcodeData is loaded):**
- Username: h3 (leetcodeData.username)
- Ranking: <p>Ranking: {leetcodeData.ranking}</p>
- Solved breakdown:
  - Easy: {leetcodeData.easy_solved} — color '#10b981'
  - Medium: {leetcodeData.medium_solved} — color '#f59e0b'
  - Hard: {leetcodeData.hard_solved} — color '#dc2626'
  - Total: {leetcodeData.total_solved}
- Acceptance rate: {leetcodeData.acceptance_rate}%

Each widget has an input + button above the data:
```jsx
<div style={{ display: 'flex', gap: '8px', marginBottom: '16px' }}>
  <input
    value={githubUsername}
    onChange={e => setGithubUsername(e.target.value)}
    onKeyDown={e => e.key === 'Enter' && loadGithub()}
    placeholder="GitHub username"
    style={{ flex: 1, padding: '8px', border: '1px solid #e2e8f0', borderRadius: '6px' }}
  />
  <button onClick={loadGithub} disabled={githubLoading || !githubUsername.trim()}
    style={{ padding: '8px 16px', backgroundColor: '#1e293b', color: 'white', border: 'none', borderRadius: '6px', cursor: 'pointer' }}>
    {githubLoading ? 'Loading...' : 'Load'}
  </button>
</div>
```

Show error in red below the input row if githubError/leetcodeError is set.
Show <p>Loading...</p> while loading and data is null.
