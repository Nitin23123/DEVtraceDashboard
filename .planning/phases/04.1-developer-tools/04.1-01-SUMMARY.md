## Decisions Made

- PG15 does not support `ADD CONSTRAINT IF NOT EXISTS` (only PG17+) — used a `DO $$` block querying `pg_constraint` instead
- Pin toggle uses `SET pinned = NOT pinned` in a single SQL statement to avoid read-modify-write race conditions
- DSA progress upsert inserts with `completed = TRUE` on first call; conflict path toggles with `NOT user_dsa_progress.completed` — first toggle always marks complete
- Destructured `{ pool, query }` from db/index.js in github.js so existing pool.query() calls continue working and new PR endpoint gets the named query() helper

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed ADD CONSTRAINT IF NOT EXISTS syntax for PostgreSQL 15**
- **Found during:** Task 1 (DB migration 003 application)
- **Issue:** `ALTER TABLE dsa_problems ADD CONSTRAINT IF NOT EXISTS` errors on PG15 (requires PG17). Migration applied but halted at constraint step.
- **Fix:** Replaced with a `DO $$ BEGIN IF NOT EXISTS (SELECT 1 FROM pg_constraint WHERE ...) THEN ALTER TABLE ... ADD CONSTRAINT ...; END IF; END$$;` block
- **Files modified:** backend/src/db/migrations/003_developer_tools.sql
- **Verification:** Re-run of migration produced only `NOTICE` (already exists) outputs, no errors. 79 rows in dsa_problems confirmed.
- **Committed in:** 8fd875b (Task 1 commit)
