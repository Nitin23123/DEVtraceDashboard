### Human Verification Required

#### 1. Dark Mode Toggle and Persistence

**Test:** Log in, click the moon icon in the nav header.
**Expected:** Page background switches to `#0f172a`, text switches to `#f1f5f9`. Refresh the page — dark mode is still active. Click the sun icon — returns to light mode.
**Why human:** CSS variable application via `document.documentElement.style.setProperty` and localStorage persistence cannot be verified without a running browser.

#### 2. Pomodoro Timer Full Cycle

**Test:** Navigate to /pomodoro. Click Start. Verify the countdown decrements every second. Click Pause — countdown freezes. Click Reset — returns to 25:00. (Optionally: allow countdown to reach 0 to verify auto-switch to break mode and browser notification.)
**Expected:** Timer decrements correctly. After reaching 0 in work mode: session count increments to 1, mode switches to "Break Time", countdown resets to 5:00. Browser notification fires if permission was granted.
**Why human:** setInterval behavior, Notification API permission request, and auto-mode-switch at boundary require a live browser environment.

#### 3. Snippets CRUD End-to-End

**Test:** Navigate to /snippets. Click "+ New Snippet", fill in title "Test", language "Python", code `print("hello")`. Click Create Snippet. Verify it appears in the list. Click Expand — code block shows. Click Copy — paste elsewhere to confirm clipboard content. Click Edit — change title, save. Click Delete — confirm prompt, snippet removed.
**Expected:** All CRUD operations reflect in the UI without page refresh. Copy writes exactly the snippet content to clipboard.
**Why human:** navigator.clipboard, window.confirm, and live API round-trips require a running app with DB.

#### 4. DSA Tracker Progress Persistence

**Test:** Navigate to /dsa. Verify progress bar shows 0/79. Click a day header to expand. Check a problem checkbox. Verify progress bar increments. Refresh the page — verify the checkbox is still checked.
**Expected:** Checkbox state persists across refresh (stored in user_dsa_progress table). Progress bar accurately reflects completed count.
**Why human:** Requires a running DB with the migration applied and an active user session.

#### 5. Task Pinning Sort Order

**Test:** Navigate to /tasks with at least 2 tasks. Click the 📌 button on a non-top task. Verify it moves to the top of the list with full-opacity pin icon. Click again — task moves back down with dimmed pin icon.
**Expected:** Pinned tasks appear before unpinned tasks. Sort is stable (newest-first within each group).
**Why human:** Visual sort behavior and opacity change require live UI interaction.

#### 6. GitHub PR Queue (Requires Connected Account)

**Test:** With a GitHub account connected, navigate to /profile. Scroll to the bottom of the profile page.
**Expected:** "Open Pull Requests" section is visible. If the account has open PRs, each shows a title link, repo badge (owner/repo), date, and any labels. If no open PRs, shows "No open PRs found."
**Why human:** Requires GitHub OAuth connected account to trigger the conditional render and the /api/profile/github/prs API call.
