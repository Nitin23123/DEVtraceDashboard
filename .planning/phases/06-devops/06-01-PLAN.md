    Create the directory `.github/workflows/` and write `ci.yml` with the following exact specification:

    Trigger: `on: push` targeting branch `main` only (`branches: [main]`). Also add `on: pull_request` targeting main so PRs are also checked.

    Job name: `build` running on `ubuntu-latest`.

    Steps:
    1. `actions/checkout@v4` — check out source
    2. `docker build -t devtrackr-backend ./backend` — build backend image (uses backend/Dockerfile which runs `npm install`)
    3. `docker build -t devtrackr-frontend ./frontend` — build frontend image (uses frontend/Dockerfile which runs `npm install --legacy-peer-deps` then serves via react-scripts)

    Do NOT add any steps that require secrets (no docker push, no deploy, no registry login). The CI job is a build check only.

    Do NOT add `POSTGRES_DB`, `JWT_SECRET`, or any `.env` values to the workflow — the Docker build steps do not need them (they only install dependencies and copy files; they do not run the app).

    The workflow file should look like:

    ```yaml
    name: CI

    on:
      push:
        branches: [main]
      pull_request:
        branches: [main]

    jobs:
      build:
        runs-on: ubuntu-latest

        steps:
          - name: Checkout code
            uses: actions/checkout@v4

          - name: Build backend Docker image
            run: docker build -t devtrackr-backend ./backend

          - name: Build frontend Docker image
            run: docker build -t devtrackr-frontend ./frontend
    ```

    Write exactly this content. No extra steps, no matrix strategy, no caching (keep it simple and fast).
  </action>
